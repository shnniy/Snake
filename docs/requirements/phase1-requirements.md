# 贪吃蛇游戏 — 第1期需求文档

## 1. 概述

### 1.1 项目背景
本项目旨在开发一款经典的贪吃蛇游戏，使用 Java 语言实现核心游戏逻辑，支持 Web 浏览器和本地 exe 两种运行方式。第1期的目标是完成核心游戏引擎的开发，确保游戏逻辑完整、正确，并通过命令行方式进行基本验证。

### 1.2 第1期目标
完成纯 Java 游戏引擎（无框架依赖），包含完整的游戏逻辑，可通过命令行启动并进行基本游玩。

## 2. 功能需求

### 2.1 网格系统
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F1.1 | 定义游戏网格，默认大小为 20×15 | P0 |
| F1.2 | 支持自定义网格大小（最小 10×10） | P1 |
| F1.3 | 提供边界检查功能，判断坐标是否在网格内 | P0 |
| F1.4 | 提供空闲格子查询功能，用于食物生成 | P0 |

### 2.2 蛇的控制
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F2.1 | 蛇身使用坐标链表表示，头部为第一个元素 | P0 |
| F2.2 | 蛇初始长度为 3 节（可配置） | P0 |
| F2.3 | 蛇的初始方向为向右（RIGHT） | P0 |
| F2.4 | 支持上/下/左/右四个方向的移动 | P0 |
| F2.5 | 每次 tick 蛇向当前方向移动一格 | P0 |
| F2.6 | 禁止蛇 180° 掉头（反向移动被拒绝） | P0 |
| F2.7 | 移动时尾部缩进（除非在增长中） | P0 |
| F2.8 | 提供 peekNextHead() 方法预览下一步位置，不修改状态 | P0 |

### 2.3 食物系统
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.1 | 食物随机生成在空闲格子上 | P0 |
| F3.2 | 食物不与蛇身重叠 | P0 |
| F3.3 | 支持普通食物（分数 10）和奖励食物（分数 30）两种类型 | P1 |
| F3.4 | 奖励食物出现概率为 10% | P2 |
| F3.5 | 吃到普通食物蛇增长 1 节，奖励食物增长 3 节 | P1 |

### 2.4 碰撞检测
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F4.1 | 检测蛇头是否撞到墙壁（网格边界） | P0 |
| F4.2 | 检测蛇头是否撞到自己的身体 | P0 |
| F4.3 | 碰撞发生后游戏结束（GAME_OVER） | P0 |

### 2.5 分数系统
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F5.1 | 吃到食物后增加对应分数 | P0 |
| F5.2 | 维护当前分数和最高分数 | P0 |
| F5.3 | 当前分数超过最高分时自动更新最高分 | P0 |
| F5.4 | 游戏重置时保留最高分 | P1 |

### 2.6 游戏状态管理
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F6.1 | 支持四种游戏状态：就绪(READY)、运行中(RUNNING)、已暂停(PAUSED)、游戏结束(GAME_OVER) | P0 |
| F6.2 | 状态流转：READY → RUNNING → PAUSED/GAME_OVER → READY | P0 |
| F6.3 | 支持游戏开始(start)、暂停(pause)、恢复(resume)、重置(reset)操作 | P0 |

| F7.3 | tick() 是唯一的状态修改方法，保证线程安全 | P0 |
| F7.4 | 在 tick 开始时读取用户输入（方向变更、加速状态） | P0 |

### 2.8 加速功能

## 3. 非功能需求

### 3.1 性能要求
- tick() 方法执行时间 < 1ms
- 游戏循环抖动 < 5ms
- 内存占用 < 50MB

### 3.2 质量要求
- 单元测试代码行覆盖率 > 90%
- 分支覆盖率 > 85%
- 所有公共方法必须有单元测试

### 3.3 设计约束
- 引擎层不依赖任何框架（纯 Java）
- 所有类使用 Java 17 特性（record、enum 增强等）
- 使用观察者模式通知状态变更

## 4. 验收标准

1. 所有单元测试通过（mvn test）
2. JaCoCo 覆盖率满足要求
3. 游戏引擎可通过命令行启动并正常游玩
4. 方向键控制蛇移动，Shift 加速，碰撞检测正确
5. 食物生成无异常（不在蛇身上、不越界）
