# 贪吃蛇游戏 — 第2期需求文档

## 1. 概述

### 1.1 背景
第1期已完成纯 Java 游戏引擎的开发，通过了全部 85 个单元测试。第2期的目标是将游戏引擎封装到 Spring Boot Web 服务中，通过 WebSocket 实现实时通信，并使用 HTML5 Canvas 在浏览器中渲染游戏画面。

### 1.2 第2期目标
完成 Spring Boot Web 服务 + HTML5 Canvas 前端，使用户可以通过浏览器访问和游玩贪吃蛇游戏。

## 3. 非功能需求

### 3.1 性能要求
- WebSocket 推送延迟 < 50ms
- Canvas 渲染帧率 ≥ 30 fps
- 页面加载时间 < 3 秒
- 支持至少 10 个并发游戏会话

### 3.2 兼容性要求
- 支持 Chrome/Edge/Firefox 最新版本
- 支持 1920×1080 和 1366×768 分辨率

## 4. 验收标准

1. Spring Boot 服务器成功启动在 8080 端口
2. 浏览器访问 http://localhost:8080 显示游戏页面
3. Canvas 正确渲染游戏画面
4. 方向键可控制蛇移动，Shift 加速
5. 游戏状态正确（开始/暂停/结束/重新开始）
6. WebSocket 通信正常，状态实时同步
7. 素材文件生成正确（PNG + WAV）
