# 贪吃蛇游戏 — 第3期需求文档

## 1. 概述

### 1.1 背景
第1期和第2期已完成核心游戏引擎和 Web 服务的基础功能。第3期的目标是增强持久化能力（排行榜）、优化用户体验（难度递增、完整音效）、实现 Windows 原生 EXE 打包。

### 1.2 第3期目标
完成数据持久化、EXE 打包和整体体验优化，使游戏成为一个完整的可独立运行的产品。

## 2. 功能需求

### 2.1 数据持久化（排行榜）
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.1 | 使用 H2 文件数据库持久化最高分记录 | P0 |
| F3.2 | 记录包含：玩家名称、分数、蛇长度、吃掉食物数、创建时间 | P1 |
| F3.3 | 游戏结束时自动保存分数 | P0 |
| F3.4 | 提供 REST API 查询排行榜前 10 名 | P0 |
| F3.5 | 提供 REST API 提交分数 | P0 |
| F3.6 | 游戏结束界面显示排行榜 | P0 |
| F3.7 | 排行榜第一名高亮显示 | P2 |
| F3.8 | 数据库文件持久化到磁盘（data/目录） | P0 |

### 2.2 难度递增
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.9 | 每吃 N 个食物后自动加快游戏速度（N 可配置） | P1 |
| F3.10 | 速度提升有上限（不低于 30ms tick） | P2 |

### 2.3 音效完善
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.11 | 吃食物时播放音效（Web Audio API 合成） | P1 |
| F3.12 | 游戏结束时播放音效 | P1 |
| F3.13 | 加速激活时播放音效 | P2 |
| F3.14 | 音效可通过 AudioManager 管理 | P1 |

### 2.4 EXE 打包
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.15 | 使用 jpackage 生成 Windows 原生 .exe | P0 |
| F3.16 | 提供打包脚本 package-exe.bat | P0 |
| F3.17 | 支持无 WiX Toolset 时的 app-image 备用方案 | P1 |
| F3.18 | EXE 启动后自动打开浏览器 | P2 |

### 2.5 UI 优化
| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F3.19 | 排行榜界面美观大方 | P2 |
| F3.20 | 加速视觉效果（蛇发光 + 速度图标） | P1 |
| F3.21 | 连接状态实时显示 | P1 |
| F3.22 | 响应式布局适配不同分辨率 | P2 |

## 3. 验收标准

1. 游戏分数持久化保存，重启后排行榜不丢失
2. 排行榜 API 正常工作（查询/提交）
3. 游戏结束界面显示排行榜
4. jpackage 成功生成 EXE 或 app-image
5. EXE 可正常启动并访问 Web 界面
6. 全部 89+ 测试通过
